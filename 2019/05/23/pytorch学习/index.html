numpy中ravel() & flatten()的区别

ravel()和flatten()都是把一个矩阵张成一个向量，其中ravel()返回的向量是一个类似引用的东西，修改这个会影响原矩阵；而flatten()则是相当于copy了一份，修改这个不会影响原矩阵。

permute transpose view reshape

首先说一下pytorch底层是如何存储这些tensor的，pytorch底层是C实现的，而C在在存储多维数组采用的是行优先展开数组的形式，通过flatten()展开的一维数组形式与实际存储形式一致。

reshape的操作方法是将读取tensor的一维形式（实际定义可能是多维的，即flatten展开）按照顺序，reshape成规定的样子（虽然未修改原tensor形状，但新建了一份tensor元信息，在其内重新指定stride）相当于tensor.contiguous().view()

torch.view等方法操作需要连续的Tensor。
transpose、permute 操作虽然没有修改底层一维数组，但是新建了一份Tensor元信息，并在新的元信息中的 重新指定 stride。torch.view 方法约定了不修改数组本身，只是使用新的形状查看数据。如果我们在 transpose、permute 操作后执行 view，Pytorch 会抛出以下错误

如果我们是按照tensor内部的stride跳着访问得到的一个tensor，则该tensor在内存上连续，在语义上不连续，view的工作环境是在底层数组上按指定的形状reshape，即使view不报错，得到的结果还是原先tensor的结果

clone detach

clone返回一个和源张量同shape、dtype和device的张量，与源张量不共享数据内存，但提供梯度的回溯

detach的机制则与clone完全不同，即返回一个和源张量同shape、dtype和device的张量，与源张量共享数据内存，但不提供梯度计算，即requires_grad=False，因此脱离计算图
